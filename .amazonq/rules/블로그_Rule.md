# Jekyll 블로그 프로젝트 Rule

## 기본 규칙
- 한국어로 답변한다.
- 파일 수정 전에 반드시 현재 내용을 먼저 읽고 확인한다.
- 불확실한 내용은 추측하지 않고 "확인이 필요하다"고 명시한다.
- 사용자의 기존 코드를 임의로 삭제하지 않는다.
- 변경 사항이 생기면 `블로그_히스토리.md`의 최상단에 변경 이력을 추가한다.

## 포스트 작성 규칙
- 새 포스트 작성 시 front matter에 다음 필드를 반드시 포함한다:
  - `permalink`: 영문으로 작성 (한글 URL 금지)
  - `title`: `[카테고리] 제목` 형식
  - `description`: 50~160자, 핵심 키워드 포함
  - `tags`: 관련 키워드 3~8개
  - `toc: true`, `toc_sticky: true`
  - `last_modified_at`: 작성/수정 날짜
- 포스트 파일명은 `YYYY-MM-DD-제목.md` 형식을 따른다.
- 본문 제목은 `## ✅` (대제목), `### 📌` + `<hr>` (소제목) 형식을 사용한다.
- 문체는 `~이다/~한다` 체를 사용한다 (~합니다 체 사용 금지).
- 여백은 `<br>` 태그 대신 CSS margin으로 처리한다 (단독 br 줄 금지).
- 이미지는 `/img/` 폴더에 저장하고 상대경로로 참조한다 (GitHub raw URL 금지).
- 관련 포스트가 있으면 `> 📚 관련 포스팅` 블록으로 내부 링크를 추가한다.

## SCSS/CSS 규칙
- SCSS 파일에 한글 주석 사용 시 파일 상단에 `@charset "utf-8";`을 반드시 포함한다.
- 반응형 스타일 수정 시 모바일($small: 600px 이하)을 반드시 고려한다.

## 빌드/배포 규칙
- 빌드 명령어: `RUBYOPT="-EUTF-8" bundle exec jekyll serve`
- 파일 수정 후 빌드 테스트를 실행하여 경고/에러가 0개인지 확인한다.
- Liquid 템플릿에서 한글 카테고리는 `site.categories["카테고리명"]` 형식을 사용한다.

## SEO / 구글 검색 노출 규칙

이 블로그의 모든 포스트는 구글 검색 노출을 최대화하는 방향으로 작성한다.

### 타이틀 최적화
- `title`에 핵심 키워드를 반드시 포함한다.
- 타이틀 길이는 공백 포함 50~60자 이내로 작성한다 (구글 검색 결과에서 잘리지 않도록).
- "총정리", "완벽 가이드", "사용법" 등 검색 의도에 맞는 단어를 포함하면 클릭률이 올라간다.

### description 최적화
- 50~160자로 작성하고, 핵심 키워드를 앞쪽에 배치한다.
- 포스트마다 고유한 description을 작성한다 (중복 금지).
- "알아보다", "정리한다" 등 사용자가 검색할 법한 자연스러운 문장으로 작성한다.

### 본문 구조 최적화
- 제목 계층을 순서대로 사용한다 (H2 → H3 → H4, 단계를 건너뛰지 않는다).
- 첫 번째 단락(H2 직후)에 핵심 키워드를 자연스럽게 포함한다.
- 단락은 3~5문장 이내로 짧게 작성한다 (가독성 향상).
- 표, 리스트, 코드 블록 등 다양한 형식을 활용하여 체류시간을 늘린다.
- 콘텐츠 길이는 최소 1500자 이상을 권장한다 (구글은 상세한 콘텐츠를 선호한다).

### URL 최적화
- `permalink`는 영문 소문자 + 하이픈으로 작성한다 (예: `/spring-boot-crud/`).
- URL에 핵심 키워드를 포함한다.
- URL은 짧고 직관적으로 작성한다 (날짜, 특수문자 지양).

### 이미지 최적화
- 모든 이미지에 의미 있는 `alt` 텍스트를 작성한다 (구글 이미지 검색 노출 대상).
- 이미지 파일명에 키워드를 포함한다 (예: `spring-boot-crud.png`).
- 이미지 크기는 가로 1200px 이하, 용량 500KB 이하를 권장한다.

### 내부 링크
- 관련 포스트가 있으면 반드시 내부 링크를 추가한다 (`> 📚 관련 포스팅` 블록).
- 본문 중간에도 관련 개념이 나오면 해당 포스트로 링크를 건다.
- 내부 링크는 구글이 사이트 구조를 이해하는 데 도움을 주고, 체류시간을 늘려 SEO에 유리하다.

### 콘텐츠 품질 (E-E-A-T)
- 구글은 **경험(Experience), 전문성(Expertise), 권위성(Authoritativeness), 신뢰성(Trustworthiness)**을 중시한다.
- 직접 경험한 내용, 실제 코드 예시, 실무 활용 사례를 포함하면 품질 평가가 높아진다.
- "이런게 있다" 수준의 나열이 아니라, "이렇게 쓰는 것이다"라는 실용적 내용을 담는다.
- 출처가 있는 정보는 공식 문서 링크를 함께 제공한다.

### 최신성 유지
- 포스트 수정 시 `last_modified_at`을 반드시 업데이트한다.
- 구글은 최근에 업데이트된 콘텐츠를 선호한다.
